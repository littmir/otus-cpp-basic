# Структуры и классы
## Парадигмы С++:
- процедурная;
- объектно-ориентированная;
- обобщённое программирование;
- метапрограммирование.  
---
Если функция-член не меняет состояние полей класса, то необходимо аннотировать ей как const.
```C++
class SomeClass {
  void some_func() const;
}
```
Если описаны конструкторы с аргументами, то компилятор больше сам не будет генерировать конструктор по умолчанию без аргументов, и такой конструктор нужно будет описать явно.  
Если конструктор по умолчанию необходим, можно использовать ключевое слово *default*. Тогда компилятор сгенерирует конструктор по умолчанию, который ничего не делает.
```C++
class Point {
  Point() = default;
  Point(int x, int y) : x(x), y(y) {}
}
```
Конструктор по умолчанию делает **default initialization**, который для сложных типов представляет собой вызов их конструктора по умолчанию.  

Все константные поля должны быть инициализированы через **список инициализации**.

Поля можно **инициализировать на месте** при объявлении значением по умолчанию. Но лучше для этого использовать конструктор и список инициализации. Инициализация в списке инициализации переписывает беззатратно инициализацию в классе.
```C++
struct Point {
  int x{5};
  int y{5};
  std::string name {"default name"};
}
```
Вызов **delete** проверяет сам, является ли указатель nullptr.
